# 报告输出格式

按解读深度与政策复杂度三选一。表格中未从原文获取到的字段填「未提及」，不要臆造。

## 格式 A：深度解读报告

用于重要政策、复杂法规，或用户明确要求影响评估时。

```markdown
# {政策名称} — 政策解读

## 一句话概要

[用最精炼的语言概括政策核心意图和影响]

## 政策基本信息

| 项目 | 内容 |
|------|------|
| 发文机关 | ... |
| 文号 | ... |
| 发布日期 | ... |
| 实施日期 | ... |
| 政策层级 | 法律 / 行政法规 / 部门规章 / 规范性文件 |
| 适用范围 | ... |

## 核心要点解读

### 要点一：{标题}

- 政策原文要旨：...
- 解读说明：...
- 合规要求：...

### 要点二：{标题}

- 政策原文要旨：...
- 解读说明：...
- 合规要求：...

## 关键时间节点

| 时间 | 事项 | 要求 |
|------|------|------|
| ... | ... | ... |

## 处罚与法律责任

| 违规情形 | 处罚措施 | 责任主体 |
|----------|----------|----------|
| ... | ... | ... |

## 企业影响评估

### 直接影响

- ...

### 需采取的合规措施

1. ...
2. ...

### 建议时间表

- 短期（1-3 个月）：...
- 中期（3-6 个月）：...
- 长期（6 个月以上）：...

## 关联政策

- [相关政策名称](链接) — 关联说明

## 信息来源

- [政策原文标题](链接) — 官网直连 / 网页搜索，发布日期
```

## 格式 B：要点速览

用于快速查询或条款简单的政策。

```markdown
# {政策名称} 要点速览

发文机关：{部委} | 文号：{文号} | 实施日期：{日期}

政策目的：

> 一段话总结政策制定的背景和目的

核心要点：

1. {要点 1}（合规要求说明）
2. {要点 2}（合规要求说明）
3. {要点 3}（合规要求说明）

企业须知：

- 必须做：...
- 禁止做：...
- 截止期限：...

处罚风险：{简要说明违规后果}

来源：[链接] — 官网直连 / 网页搜索
```

## 格式 C：多政策对比

用于用户关注多项政策的差异或演变时。

```markdown
# {领域} 政策对比分析

| 维度 | {政策 1} | {政策 2} | {政策 3} |
|------|---------|---------|---------|
| 发文机关 | ... | ... | ... |
| 实施日期 | ... | ... | ... |
| 核心要求 | ... | ... | ... |
| 适用范围 | ... | ... | ... |
| 处罚力度 | ... | ... | ... |

政策演变趋势：...

合规建议：...

信息来源：逐条列出链接与来源层级
```
