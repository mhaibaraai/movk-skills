# 模板速查

5 套真实设计模板，注册表见 [templates/registry.json](../templates/registry.json)，页型索引见 `templates/index/`。
产出页面是模板原页的深拷贝，配色、字体、装饰形状、配图均来自模板本身，不存在「主题色配置」这回事。

| 模板键 | 总页数 | 页型库存 | 内容页可容纳的要点数 |
|--------|--------|----------|---------------------|
| `员工安全知识培训` | 25 | 封面1 目录1 章节4 内容18 封底1 | 3 / 4 / 5 / 6 |
| `项目立项汇报模版` | 30 | 封面1 目录1 章节6 内容18 表格3 封底1 | 4 / 5 / 6 |
| `年终安全工作汇报模版` | 25 | 封面1 目录1 章节5 内容17 封底1 | 4 / 5 / 6 |
| `重大事件专项汇报` | 41 | 封面1 目录1 章节6 内容32 封底1 | 2 / 3 / 4 / 5 / 6 |
| `通用模板` | 55（精修） | 封面1 目录1 章节4 内容48 封底1 | 2 / 3 / 4 / 5 / 6 |

`通用模板` 用于任意主题的通用汇报/方案，无固定场景时兜底；源稿 95 页（讲解+版式混排），已精修为 55 页可复用集。

「页型库存」是可被克隆的源页数量，同一张源页可被重复克隆（章节页会自动改写序号），
所以总页数不受模板原页数限制——20 页的成品可以只用到 8 张不同的源页。

## 场景别名

| 用户说法 | 落到的模板 |
|----------|-----------|
| 安全培训 / 安全生产培训 / 危化品 / 危化品管理 / HSE 培训 / 政策宣贯 | `员工安全知识培训` |
| 项目申报 / 立项 / 课题答辩 / 项目汇报 / 申报评审 | `项目立项汇报模版` |
| 年终总结 / 年度总结 / 工作汇报 / 安全汇报 / 工作总结 | `年终安全工作汇报模版` |
| 专项汇报 / 事故汇报 / 应急事件 / 重大事件 / 事故调查 | `重大事件专项汇报` |
| 通用 / 自定义 / 其他 / 不限 / 任意主题 | `通用模板` |

别名与默认章节都在 `registry.json` 里改，改完要同步平台工作流主对话节点的系统角色里列出的模板清单。
模板一律以用户选定值为准；若推断场景与选定模板明显不符，应向用户确认是否切换，而非擅自改判。

## 页型说明

| kind | 用途 | 可回填的槽位 |
|------|------|-------------|
| `cover` | 封面 | `title`、`subtitle` |
| `toc` | 目录 | 各章节名（`item_head`） |
| `section` | 章节分隔页 | `title`；序号槽 `no` 由脚本按章节顺序自动写 01/02… |
| `content` | 内容页 | `title`（页眉，通常填章节名）、若干条 `item_head` + `item_body` |
| `table` | 表格页 | 仅 `项目立项汇报模版` 有，当前渲染器只回填其文本槽 |
| `closing` | 封底 | `title` |

`decor` 槽（`LOGO`、`PART 01`、`PROJECT APPLICATION`、内容页自带的 01/02 条目编号）一律保持原样，不回填。

## 套用用户自带模板

用户上传 .pptx 时不必改注册表，一条命令取回并索引成临时模板：

```bash
uv run skills/create-ppt/scripts/import_template.py "/api/file/xxx" --base "https://平台域名" --out-dir tpl --name 我的模板
```

来源可以是平台相对路径（配 `--base`）、完整 URL 或本地文件。产出 `tpl/index.json`（版式索引 +
从章节页自动提取的默认 `sections`）与 `tpl/source.pptx`。之后所有命令加 `--index tpl/index.json`
（`build_pptx` / `check_pptx` 另加 `--source tpl/source.pptx`）即可旁路注册表，用法与内置模板完全一致。

索引用启发式推断页型与槽位角色（`kind` / `role` / `items` / `cap`），产物是**草稿**——
先跑 `thumbnail.py` 出缩略图核对封面、目录、章节页有没有被误判，比逐条读 JSON 快得多。

沙箱每轮全新、索引不跨轮存在：修改轮与渲染轮都要按同一来源重跑 `import_template.py`
（索引是确定性的，重跑得到同一份，卡片重建才对得上）。

`registry.json` 注册仍适用于要**长期内置**的模板：把 pptx 放进 `assets/`、索引放进 `templates/index/`，
再加一项 source / index / aliases / sections。
