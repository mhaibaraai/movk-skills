# 企业 PPT 写作规范

## 首要原则：内容适配版式，而不是版式迁就内容

页面来自模板已设计好的版式，槽位数量和字数容量都是固定的。
骨架每页都带一个 `caps` 字段，写死了该页各类槽位的字数上限——**不要增删条数，不要超字数**。写不下就加页，不要挤。

`caps` 是按文本框实际宽高与字号量出来的，不是建议值。超了不是「稍微挤一点」：
15 字的标题塞进 6 字的封面框，会挤成三行盖住整个版面，整页作废。

- `title`（页眉/主标题）通常填章节名，与目录保持一致。封面主标题往往只有 6–10 字的空间，
  项目全称这类长名称拆到副标题里去。
- `item_head`（要点标题）动宾结构、可量化：「可燃气体积聚」优于「关于气体的问题」。
- `item_body`（要点说明）一句话讲清做什么、到什么程度，带上数字与标准。
- 封面/封底的署名槽（汇报人、汇报单位、日期）按实际信息填，别留模板原文。

## 通用要求

- 纯中文，术语规范（安全场景如「危化品」「HSE」「动火作业」「八大特殊作业」「爆炸下限」）。
- 每页单一主题，结论先行——先给判断，再给依据。
- 不编造数据。事故数、投资额、检测值等缺失时，按下面「联网取数与来源标注」处理：
  要么向用户确认，要么取带出处的公开行业数据，不要填「XX」占位、更不要自己编一个像样的数字。
- 数字与单位半角，中英文之间加空格：「低于爆炸下限的 20%」「巡查 30 分钟」。

## 直接文本/大纲路径

用户上传文档或粘贴成段文稿/大纲时，跳过意图确认，直接以其内容为素材填充所选模板骨架：
- 提炼原文的层级结构对应到章节与要点，压缩到每页 `caps` 之内；不照抄长段。
- 原文缺章节时按模板默认 `sections` 归拢；原文自带结构时以原文为准。
- 只做提炼与改写，不新增原文没有的数据或结论。

## 联网取数与来源标注

用户只给主题、却要求含真实数据（「含事故数据」「带同比数字」）时，手上没有任何数据源。
此时先在意图卡追加第 6 项「数据来源」让用户选定，确认走公开行业数据后再用 `web-fetch` 检索。

检索顺序按 web-fetch 自身规范：`sitemap.py` 枚举 → 回落 `search.py` 关键词 → `fetch.py` 取正文。

政务站实测（2026-07）：`mem.gov.cn` 未声明 sitemap，`search.py` 走的 360 又常因出口 IP 被拦而报
`blocked`——**这两条通道对权威源经常双双不通，属常态而非异常**。此时走 web-fetch 的第三条路：
抓入口页、顺锚文本逐级定位（`mem.gov.cn` 首页 http 层可直接抓通，栏目锚文本完整）。

```bash
uv run skills/web-fetch/scripts/fetch.py --urls '["https://www.mem.gov.cn/"]' --links
```

从 `links[].text` / `attachments[].text` 的**锚文本**认目标，再抓那条 URL。绝不按 URL 命名规律猜——
政务站的 URL 多是拼音缩写（ndbg/qyshzrbg），单看 URL 分不出哪份是目标。

- **只取权威公开源**：应急管理部、国家统计局、行业协会年报、监管通报。不取自媒体、二手转述、
  问答站——PPT 上的数字要经得起追问。
- **每条数据必须带短来源标注**，且标注计入该槽位 `caps` 字数。写不下就换更短的数据表述，
  不能靠删来源腾字数——溢出会把整页版式压垮，无出处的数字则是另一种形式的编造。
- **公开行业数据 ≠ 公司内部数据**。这是最容易越的界：行业事故总数不能写成「我司事故 N 起」。
  主语必须如实，把口径讲清楚。
- **取不到就如实说**。`errors[].kind` 为 `blocked` / `network_unreachable` 是环境限制
  （出口 IP 被拦），`no_match` 才是真无结果——两者都不等于「可以自己编一个」。
  告知用户「联网检索未取到数据」，请其上传内部台账或确认留占位。

```text
正例：全国化工事故 328 起，同比降 12%（据应急管理部 2025 年通报）
反例：我司事故 328 起，同比降 12%          — 行业数据冒充内部数据
反例：全国化工事故 328 起，同比降 12%      — 有数字无出处，等同编造
反例：事故 XX 起，同比降 XX%               — 占位符进了成品
```

内部数据永远优于公开数据：用户能提供台账/统计表时，引导其上传走文档提取通路。

## 页数分配

`make_outline.py` 自动排布：封面 1 + 目录 1 + 每章 1 张分隔页 + 正文页在章节间均分 + 封底 1。
余数摊给靠前的章节，保证总页数与用户要求一致。

| 总页数 | 建议章节 | 每章正文页 |
|--------|---------|-----------|
| 12–16 | 4 | 1–2 |
| 18–24 | 4–5 | 2–3 |
| 25–30 | 5–6 | 3–4 |

## 各场景要点

- **安全培训**：方针 → 风险辨识 → 操作规程 → 应急处置。至少一章讲事故案例，写清直接原因与整改闭环。
- **项目立项**：背景依据 → 现状问题 → 目标思路 → 建设方案 → 实施计划 → 经费效益。投资与效益用该模板的表格页量化。
- **年终汇报**：总览 → 制度体系 → 问题分析 → 不足与风险 → 下年计划。成效必须带同比数字。
- **专项汇报**：事件概述 → 应急响应 → 影响评估 → 原因性质 → 责任处理 → 防范整改。时间线要精确到时分。
- **通用模板**：无固定场景，按主题或用户大纲组织；默认 背景与目标 → 现状分析 → 实施方案 → 计划安排 → 总结展望。
