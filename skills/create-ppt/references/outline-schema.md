# 大纲 JSON Schema

`build_pptx.py` 读取的结构。所有文本纯中文。

```jsonc
{
  "template": "员工安全知识培训",   // 内置：模板键或别名（见 templates/registry.json）
                                   // 自带模板：展示标签，索引另经 --index 传入，不查注册表
  "title": "动火作业安全培训",       // 用于默认文件名
  "pages": [ /* 每页一个对象 */ ]
}
```

## 页对象

```jsonc
{
  "src": 5,              // 必填：克隆模板的第几页（由 make_outline.py 生成，不要手改）
  "kind": "content",     // cover | toc | section | content | table | closing
  "title": "风险辨识",
  "subtitle": "…",       // 仅 cover / 部分 closing
  "caps": {              // 该页各类槽位的字数上限，由脚本按版式实测写入
    "title": 9, "item_head": 8, "item_body": 55
  },
  "items": [             // 条数由模板该页的容量决定，不得增删
    { "head": "可燃气体积聚", "body": "受限空间易积聚可燃气体，动火前须气体检测" }
  ]
}
```

| kind | 用到的字段 |
|------|-----------|
| `cover` | `title`、`subtitle`、`items[].body`（署名槽：汇报人 / 汇报单位 / 日期） |
| `toc` | `items[].head`（各章节名） |
| `section` | `title`（序号由脚本自动写） |
| `content` | `title`（页眉，通常是章节名）、`items[].head` 与/或 `items[].body` |
| `closing` | `title`、`subtitle`、`items[].body`（署名槽） |

封面/封底的 `items` 占位会带上模板原文提示，如 `{"body": "{汇报人：待用名}"}`——按实际署名信息替换。

## 硬约束

- **`src` 不可臆造**：必须是 `make_outline.py` 排布出来的模板页号，否则渲染直接报错。
- **`items` 条数与字段不可增删**：骨架给几条就写几条，且只填骨架已有的字段。有的版式一条只给
  `body`（无 `head` 槽）或只给 `head`——别自作主张补上另一个字段，多写的字段无槽可落会被静默丢弃。
- 多写的会被丢弃，少写的会留下空框。
- **字数不可超 `caps`**：这是按文本框实际宽高与字号实测出来的容量，不是建议值。
  超了不是「稍微挤一点」——15 字的标题塞进 6 字的封面框会挤成三行盖住整个版面。
- 需要更多要点时不要往一页里塞，改为增加页数让 `make_outline.py` 重新排布。
- 渲染后必须跑 `check_pptx.py`，它会按同一套 `cap` 判据把溢出全部揪出来。
- **结构先于内容**：`src` / `kind` / `caps` / `items` 条数是骨架结构，由脚本掌控；模型只填文本槽。
  文本改动走 `--patch`，页数增减、章节增减或重排这类**结构性改动重跑 `make_outline.py`** 生成新骨架再填充。
  顺序反过来（先把文案写满、再调结构）会让已写好的文案错位到别的槽位里。

## 自带模板

用户上传的 .pptx 经 `import_template.py` 索引后，同一份 schema 照常适用，只有两点不同：

- `template` 是**展示标签**（如「我的模板」），不是注册表键——重建与渲染时不查 registry。
- 所有命令要加 `--index tpl/index.json`（`build_pptx` / `check_pptx` 另加 `--source tpl/source.pptx`），
  否则会拿这个标签去注册表里找、直接报「未知模板」。

沙箱每轮全新、索引不跨轮存在：修改轮与渲染轮都要先按同一来源重跑 `import_template.py`
（索引是确定性的，重跑得到同一份，卡片重建才对得上）。

## 渲染行为

- 每页都是模板对应源页的深拷贝，同一源页可被多次克隆。
- `decor` 槽（LOGO、PART 01、条目编号）保持原样；`no` 槽仅在 `section` 页被改写为章节序号。
- 没有对应内容的可填槽位会被清空——宁可留空框，也不把「单击此处添加文本」带进成品。
- 文本槽只写**纯文本**，不要用空格或换行凑排版：回填只保留首个 run 的样式并清掉多余段落，
  靠空格做的缩进与对齐在成品里必然错位——间距、对齐、字号都是模板版式的事，不归文案管。
